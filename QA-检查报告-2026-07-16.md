## 2. 核心功能代码审查（结论：稳健）

- **请求体限流 + 预算转发**（`internal/proxy/handler.go`）
  - nginx `/v1/` 维持 32MB 通道天花板；Go 按每用户 `max_body_size` 执行 `MaxBytesReader`。
  - `compaction=trim`（默认）超预算时 `compactMessagesToBudget` **迭代式**按真实字节校验裁剪（保留全部 system + 最近轮次），至少保留 1 条最新非 system 消息；缺失/非数组/空 messages 原样返回。
  - `compaction=off` 回退旧行为（超限即 413）。超 32MB 天花板仍 413，文案为通用「请求操作已超过模型最大请求限制」，**无数值泄漏**。
- **配额扣减**（`internal/models/quota.go` → `AtomicDeductQuota`）：单条 `UPDATE ... WHERE used+? <= limit` + `RowsAffected==1`，原子 check-and-set 正确，无超扣。
- **时段路由**（`internal/router/selector.go`）：`ResolveProvider` 命中即走、绝不回退默认（5h窗口/星期 + Asia/Shanghai 时区）；`atomic.Value` 快照热加载，Admin CRUD 后 `Reload()` 原子替换；`RewriteModel` 大小写不敏感回退。
- **密钥安全**（`internal/security/encrypt.go`）：AES-256-GCM，KEK 由 `GATEWAY_KEK_ENV` 环境变量派生（SHA-256），明文 Key 仅驻内存、不落盘/不打印；`MaskKey` 脱敏正确。
- **用户管理**（`internal/admin/users.go`）：创建/更新对 `max_body_size` 均 clamp 到 [1MB, 32MB]；`fixed_multiplier` 校验 0.1–100.0；重复用户名返回 409；管理员不可改路由/不可过期/不可删；审计日志齐全。
- **鉴权中间件**（`internal/auth/middleware.go`）：`SubKeyAuth` 校验 Bearer/sub-key 前缀/哈希/禁用/过期；`max_body_size` 注入 context 供 handler 使用。
- **流式**（`internal/proxy/stream.go`）：SSE 逐行透传、model 回写、`[DONE]` 前提取 usage、客户端断连检测、10 分钟整体超时，完整。

## 5. 发现的问题与处理

### 🔴 已修复 — 仓库 nginx 配置缺陷（配置漂移）
- **现象**：`deploy/nginx.conf` 的 `location /v1/` 仍含 `limit_req zone=api_limit burst=20 nodelay;`，但文件顶部 `api_limit` 的 `limit_req_zone` 定义是**注释掉的** → 引用了**未定义**的 zone。该文件当前是**非法 nginx 配置**（`nginx -t` 会失败）。
- **根因**：2026-07-13 决策（纯中转、上游自带限速）已手动从线上移除 `/v1/` 的 `api_limit` 限速，但仓库模板未同步 → 漂移。若有人基于 `deploy/nginx.conf` 重新部署，该限速会被**重新加回**，且因 zone 未定义 nginx 会启动失败。
- **修复**（仅本地文件，未触碰线上）：
  - `deploy/nginx.conf`：删除 `/v1/` 内 `limit_req zone=api_limit` 及孤儿 `limit_req_status 429;`/`error_page 429 @ratelimit_json;`（`@ratelimit_json` 命名 location 仍被 `/m-7xa2/` 的 `login_limit` 复用，保留）；顶部注释改为"api_limit 已移除、勿再加回"。现配置**自洽、可安全部署**。
  - 文档同步：`AGENTS.md`、`docs/adr/0003-nginx-fronting-gateway.md` 中仍描述已移除的 `/v1/ api_limit` 限速与"请求体上限 1MB"，已改为" `/v1/` 无 nginx 限速、32m 通道天花板"正确表述。

### 🟡 建议（低优先级，未改）
1. **日志隐私**（`handler.go:226-231`）：JSON 解析失败时 `log.Printf("DEBUG: Raw body...")` 打印最多 500 字节原始 body（含用户对话内容，非 API Key）。生产环境建议去掉或加 debug 开关。
2. **前端档位标签**：选项标"500 KB"实际存 524288 字节（=512 KiB / 0.5 MiB），列表 `formatBodySize` 会显示"512 KB"。标签与显示不一致但无害（cosmetic）。
3. **compaction 枚举**：配置接受任意非 `off` 字符串按 `trim` 处理（无严格枚举校验），无功能危害。
4. **离散档位**：前端下拉仅 0.5/1/4/8/16/32MB，API 直设其他值（如 2MB）时编辑下拉会空白（不影响底层存储）。
