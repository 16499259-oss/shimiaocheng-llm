🔌 客户端配置

所有客户端通用：填入 Base URL + 你的 Key + 模型名称 GLM-5.2 即可使用。

https://ai.shimiaocheng.top/v1

💡 这是所有客户端的 Base URL，先复制备用。

Cursor
Cursor 是 OpenAI 兼容客户端，可直接自定义 Base URL。
1. 打开 Settings（设置） → Models（模型）。
2. 勾选 Override OpenAI Base URL，填入 https://ai.shimiaocheng.top/v1。
3. API Key 填页面顶部「你的 Key」框里的 sk-... 值。
4. Model 填 GLM-5.2。
5. 点击 Verify 验证通过后即可对话。
Claude Code
⚠️ 注意兼容性：Claude Code（Anthropic 官方 CLI）的原生协议是 Anthropic Messages API，不是 OpenAI 兼容协议。本网关仅提供 OpenAI 兼容接口，因此无法在 Claude Code 中「直接」填 Base URL 使用。

可行方案（二选一）：
方案 A — 若你的 Claude Code 版本支持自定义 OpenAI 兼容 Provider：按该 Provider 配置填入 Base URL https://ai.shimiaocheng.top/v1、API Key 填 sk-...、模型填 GLM-5.2。
方案 B（推荐）— 经 OpenAI↔Anthropic 转换代理中转：用 litellm 等代理把本网关的 OpenAI 接口转成 Anthropic 协议，然后把 Claude Code 的 ANTHROPIC_BASE_URL 指向该代理、ANTHROPIC_API_KEY 填你的 sk-... 值。
请先确认你的 Claude Code 版本是否支持 OpenAI 兼容 Provider，否则必须使用方案 B 中转。
Trae
Trae 支持 OpenAI 兼容的自定义模型Provider。
1. 打开 Settings（设置） → Model Provider（模型供应商）。
2. 选择 Custom / OpenAI Compatible 自定义 Provider。
3. Base URL 填 https://ai.shimiaocheng.top/v1。
4. API Key 填页面顶部「你的 Key」框里的 sk-... 值。
5. Model 填 GLM-5.2，保存后即可选用。
OpenCode
OpenCode 是开源 AI 编码 agent，通过配置文件接入 OpenAI 兼容 Provider。
1. 编辑配置文件 ~/.config/opencode/opencode.json（不存在则新建）。
2. 在 providers 中新增一个 openai-compatible 类型的 Provider：
baseUrl 填 https://ai.shimiaocheng.top/v1；
apiKey 填页面顶部「你的 Key」框里的 sk-... 值；
在 models 里添加一项名称为 GLM-5.2。
3. 重启 OpenCode，选择该 Provider 与 GLM-5.2 模型即可。
Hermes

1. 打开客户端的 Settings / Preferences。
2. 找到 API / Provider 配置项，选择 OpenAI Compatible 或自定义 Provider。
3. Base URL 填 https://ai.shimiaocheng.top/v1。
4. API Key 填页面顶部「你的 Key」框里的 sk-... 值。
5. Model 填 GLM-5.2。
以上为通用步骤，具体路径以实际客户端界面为准。

ZCode
ZCode 是 Z.ai（智谱）官方的 AI 编码开发工具，支持添加「自定义供应商」走 OpenAI 兼容协议接入本网关：
1. 打开 ZCode，点击左下角 模型选择器 → 管理模型 → 模型设置。
2. 点击 添加自定义供应商，协议类型选择 OpenAI 兼容。
3. API 基础 URL 填 https://ai.shimiaocheng.top/v1。
4. API Key 填页面顶部「你的 Key」框里的 sk-... 值。
5. 模型名称 填 GLM-5.2。
6. 打开该供应商的启用开关并保存。
7. 回到对话框，选中 GLM-5.2 发一句话测试连通性。
ZCode 自定义供应商仅支持 OpenAI 兼容 / Anthropic 两类协议，本网关请选「OpenAI 兼容」。
